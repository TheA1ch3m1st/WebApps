About:

This project's frontend has been made using Mobirise, as I completely suck with frontend. I coded all of the backend of course, but i had to use a website designer for the frontend, or else my website would look like it traveled through a time machine from 1993. I don't want this section to get too personal, so I will talk a little about my project now. It's a flask webapp that uses a database full of malicious files' hashes in order to check for malicious files. Of course it doesn't have any engines, but it has a huge database of malicious hashes. You upload the file, backend takes it and gets hash, then compares it to any of the hashes in the database. If anything matches, file is marked as malicious. Pretty simple, eh? The hashes that are being used are MD5 hashes from VirusShare. This project has been created for the CS50 Final Project task at the end of the course. I was going to make a CLI python password manager, but that would've been a really redundant idea and it wouldn't be very practical at all. So I decided to learn webdev (mainly backend cuz frontend sux), and here I am! I actually used to hate the idea of learning webdev but I really liked backend. I thought it would be redundant at first, but then I got into security and I realized that I need to learn how to read web code. I don't think I will ever really get into writing frontend, but I can certainly read it. Anyway, in the check route of the page, you'll find a test file that can be used to test if the web application is working. Of course getting hands on the actual malware samples would be dangerous, so I made a test file and put it's MD5 hash in the database along with the actual malware hashes.
